### Python & General Programming

**1. How does Python's garbage collection work?**  
Python uses reference counting and a cyclic garbage collector to manage memory automatically. When an object's reference count drops to zero, it's deallocated. The cyclic collector finds and frees reference cycles.

**2. What are Python decorators and why use them?**  
Decorators are functions that modify the behavior of other functions or classes. They allow code reuse, logging, access control, or modification of inputs/outputs without changing the original code.

**3. Explain the difference between `@staticmethod` and `@classmethod`.**  
- `@staticmethod`: No access to class/instance (just a function in a class).  
- `@classmethod`: Receives the class as the first argument, can modify class state.

**4. What is a generator in Python?**  
A generator is a function that yields values one at a time using the `yield` keyword. It is memory-efficient for large datasets and supports iteration without storing entire sequences in memory.

**5. How do you handle exceptions in Python?**  
Use try-except blocks to catch exceptions. Optionally add `finally` for cleanup. Use specific exception types to handle known error conditions gracefully.

### Django

**6. What is Django ORM?**  
Django ORM (Object-Relational Mapping) maps Python classes (models) to database tables. It allows querying, updating, and managing data using Python code without writing SQL.

**7. How do you create a new model in Django?**  
Define a new class in `models.py` inheriting from `django.db.models.Model`, add fields, and run migrations (`makemigrations`, `migrate`) to create corresponding tables.

**8. What are Django signals?**  
Signals allow decoupled applications to get notified when certain actions occur (e.g., saving an object). Pre-defined signals include `pre_save`, `post_save`, etc.

**9. How does Django handle URL routing?**  
In `urls.py`, you map URL patterns to views. Django checks these patterns for incoming requests and calls the corresponding view function/class.

**10. Explain Django middleware.**  
Middleware is a series of hooks between request and response phases. They can modify requests/responses, handle exceptions, or perform tasks like authentication across requests.

### Celery & RabbitMQ

**11. What is Celery and why use it?**  
Celery is a distributed task queue for handling asynchronous background jobs. It offloads long-running tasks from the main web server to improve responsiveness.

**12. How does Celery work with RabbitMQ?**  
Celery uses brokers like RabbitMQ to send and receive messages. RabbitMQ queues tasks which Celery workers then consume and execute asynchronously.

**13. How do you implement retries in Celery?**  
Use `retry` method inside a task with exceptions handling. Configure maximum retries and intervals. For example:  
```python
try:
    # task logic
except Exception as e:
    self.retry(exc=e, countdown=60, max_retries=3)
```

**14. How do you monitor Celery tasks?**  
Use tools like Flower or Celery's built-in monitoring to view task states, successes, failures, and worker status.

**15. How do you ensure idempotency in Celery tasks?**  
Assign a unique identifier to each task or use database/in-memory checks to record processed tasks. This prevents processing duplicates.

### Databases & SQL

**16. What is indexing and why is it important?**  
Indexes improve query performance by allowing faster data retrieval. They work like a book index, but too many can slow down writes.

**17. How do you optimize a slow SQL query?**  
Analyze the query plan, add proper indexes, refine SELECT clauses, avoid unnecessary joins, and optimize database schema if needed.

**18. Explain ACID properties.**  
ACID stands for Atomicity, Consistency, Isolation, Durability. They ensure reliable transactions: all-or-nothing, valid data, separation between transactions, and durability after commit.

**19. What is database normalization?**  
Normalization organizes data to reduce redundancy and improve integrity through various normal forms (1NF, 2NF, 3NF, etc.).

**20. How do you handle database migrations in Django?**  
Use Django's built-in migrations system: run `makemigrations` to create migration files and `migrate` to apply changes to the database schema.

### Advanced System Design & Architecture

**51. How would you design a distributed caching system for a web application?**  
*Answer:*  
Design a distributed cache using tools like Redis or Memcached. Use consistent hashing to distribute keys across multiple cache nodes, implement data replication for redundancy, and set up eviction policies (LRU, TTL) to manage cache size. Ensure that cache invalidation strategies maintain data consistency with the primary database, and consider a layered cache approach for different types of data.

**52. How do you handle eventual consistency in distributed databases?**  
*Answer:*  
In a distributed system, eventual consistency means that updates will propagate to all nodes over time. To handle this, design your application to tolerate temporary inconsistencies, use versioning or conflict resolution strategies (e.g., last write wins, merge logic), and implement mechanisms like vector clocks or CRDTs for conflict-free data replication. Also, design read operations to check multiple replicas if strict consistency is required temporarily.

**53. Describe how you would set up a CI/CD pipeline for a microservices architecture.**  
*Answer:*  
Use a CI/CD tool (e.g., Jenkins, GitLab CI, GitHub Actions) to automate building, testing, and deploying microservices. Each service has its pipeline triggered by commits. The pipeline includes linting, unit/integration tests, building Docker images, and pushing them to a registry. Use container orchestration (like Kubernetes) for deployment, with steps to manage blue-green or rolling updates, automated rollbacks on failure, and integration tests in staging environments before production deployment.

**54. How would you approach designing an event-driven architecture?**  
*Answer:*  
Design services that communicate via events using a message broker (Kafka, RabbitMQ). Identify events (e.g., user registration, order placed) and design event producers and consumers. Ensure idempotency in consumers, use topics/queues to decouple services, and implement event sourcing or CQRS patterns if needed. Use schema registries to manage event formats and versioning.

**55. How do you secure microservices communication?**  
*Answer:*

### Advanced Performance & Scalability

**56. How would you design a rate-limiting mechanism for an API?**  
*Answer:*  
Implement a token bucket or leaky bucket algorithm. Use a distributed cache (e.g., Redis) to store request counts or tokens per user/IP. On each request, check the current count against limits and either serve the request or throttle it. Use sliding windows or fixed windows for counting. Configure rate limits differently for various endpoints or user tiers.

**57. What strategies would you use to optimize a system with a high read-to-write ratio?**  
*Answer:*  
Use read replicas to distribute read load, implement caching layers (Redis/Memcached), use denormalized data or materialized views for frequently accessed queries, and consider using a NoSQL database if it fits the use case. Ensure that write operations propagate to replicas and caches are invalidated or updated accordingly.

**58. Explain the CAP theorem. How does it influence system design?**  
*Answer:*  
The CAP theorem states that in a distributed data system, you can only achieve two of the following three: Consistency, Availability, and Partition tolerance. In system design, this means making trade-offs: for example, a system may choose eventual consistency over strong consistency to remain available during network partitions, guiding decisions on database and architecture choices based on requirements.

**59. How do you handle database sharding?**  
*Answer:*  
Database sharding splits data across multiple databases to improve performance and scalability. Choose a sharding key that distributes load evenly, implement a routing mechanism in your application or use middleware, ensure cross-shard operations are minimized, and plan for resharding as data grows. Monitor shards for balance and performance, and handle transactions carefully as they may span multiple shards.

**60. How would you design a logging and alerting system for microservices?**  
*Answer:*  
Centralize logs from all services using a logging agent (e.g., Fluentd, Logstash) sending data to a centralized system like Elasticsearch. Use structured logging to make analysis easier. Set up dashboards with Kibana and configure alerts with tools like Prometheus Alertmanager or Grafana based on log patterns, error rates, and performance metrics. This allows proactive monitoring and rapid response to incidents.

### 76. Explain optimistic vs. pessimistic locking in databases.
**Answer:**  
- **Optimistic Locking:** Assumes minimal conflicts. It checks data version/timestamp before committing changes. If the data changed since it was read, the transaction fails, and the user retries.
- **Pessimistic Locking:** Locks the data when reading to prevent concurrent modifications. Other transactions must wait until the lock is released, reducing conflicts but potentially reducing concurrency.
